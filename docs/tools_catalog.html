Binaries (7)

Pre-built executables shipped in the crate. Each binary is gated behind specific feature flags and compiled with cargo build --bin <name> --features <flags>.

ai_assistant_server production full
Standalone HTTP API server using std::net::TcpListener. Serves the OpenAI-compatible REST API with Bearer auth, CORS, request IDs, rate limiting, graceful shutdown, and optional TLS via server-tls.
Flags: --host --port --config --api-key --tls-cert --tls-key --dry-run
ai_assistant_cli standalone no features required
Interactive REPL for the AI assistant. Supports slash commands (/help, /model, /config, /clear, /export, /exit), conversation history, model switching, and template management.
Commands: /help /model /config /clear /export /exit
ai_test_harness testing no features required
Comprehensive test runner with categorized test suites, colored output, JSON reporting, and interactive menu. Validates all library functionality across categories (security, RAG, streaming, providers, etc.).
Flags: --all --category=<name> --list --no-color --json
kpkg_tool rag
CLI for creating, inspecting, listing, and extracting encrypted knowledge packages (.kpkg). Uses AES-256-GCM encryption with pluggable key providers (AppKeyProvider, CustomKeyProvider).
Subcommands: create --input <dir> --output <file> list <file> inspect <file> extract --input <file> --output <dir>
ai_assistant_standalone production full, server-axum
All-in-one binary combining the axum HTTP server, interactive REPL, and Butler auto-configuration. Simplest deployment: a single executable for everything.
Flags: --host --port --config --api-key --tls-cert --tls-key --repl --auto-config --dry-run
ai_cluster_node enterprise full, server-cluster
Distributed cluster node with QUIC mesh networking, CRDT state sync, and session affinity. Joins clusters via bootstrap peers and join tokens.
Flags: --node-id --port --quic-port --bootstrap-peers --join-token --data-dir --dry-run
ai_proxy production server-axum
Lightweight API gateway with round-robin load balancing, session affinity (sticky sessions via X-Session-Id), and health checks. No AiAssistant instance — pure routing with minimal overhead.
Flags: --port --backends --health-interval --api-key --dry-run

Key CLI Examples

Common commands for building and running the tools.

Start the HTTP server
# Build and run the std-thread server on port 8090
cargo run --bin ai_assistant_server --features full -- --port 8090 --api-key MY_KEY

# With TLS (requires server-tls feature)
cargo run --bin ai_assistant_server --features full,server-tls -- \
  --port 8443 --tls-cert cert.pem --tls-key key.pem
All-in-one standalone (axum)
# Server + REPL in one process
cargo run --bin ai_assistant_standalone --features full,server-axum -- --port 8090 --repl

# Auto-detect environment via Butler
cargo run --bin ai_assistant_standalone --features full,server-axum,butler -- --auto-config
Cluster deployment
# Seed node
cargo run --bin ai_cluster_node --features full,server-cluster -- \
  --node-id seed1 --port 8091 --quic-port 9001

# Join existing cluster
cargo run --bin ai_cluster_node --features full,server-cluster -- \
  --node-id worker1 --port 8092 --quic-port 9002 \
  --bootstrap-peers 10.0.0.1:9001 --join-token <TOKEN>
API gateway (proxy)
# Route to 3 backend nodes with 10s health checks
cargo run --bin ai_proxy --features server-axum -- \
  --port 8080 --backends 10.0.0.1:8090,10.0.0.2:8090,10.0.0.3:8090 --health-interval 10
Knowledge packages
# Create an encrypted .kpkg from a folder
cargo run --bin kpkg_tool --features rag -- create --input ./docs --output knowledge.kpkg --name "My KB"

# Inspect package metadata
cargo run --bin kpkg_tool --features rag -- inspect knowledge.kpkg

# Extract to a folder
cargo run --bin kpkg_tool --features rag -- extract --input knowledge.kpkg --output ./extracted
Test harness
# Run all test categories
cargo run --bin ai_test_harness -- --all

# Run only security tests with JSON output
cargo run --bin ai_test_harness -- --category=security --json

# List available categories
cargo run --bin ai_test_harness -- --list
Interactive REPL
cargo run --bin ai_assistant_cli --features full
# Type /help inside the REPL for commands
