Notas Importantes

  • Todos los Tiers 0-3 son 100% gratuitos. Solo el Tier 4 (cloud managed) tiene coste monetario.
  • Qdrant en Rust NO es embebido. El crate qdrant-client es solo un cliente gRPC. Necesitas un servidor Qdrant corriendo. El modo embebido (:memory:) solo existe en Python.
  • LanceDB es el "SQLite de vector DBs" — se enlaza directamente en el binario Rust. Sin servidor, sin Docker, sin red.
  • La migracion entre tiers es transparente via el trait VectorBackend: export_all() del backend anterior + import_bulk() al nuevo. No hay que re-calcular embeddings.
  • Latencias incluyen overhead de indice. Sin indice (Tier 0), la latencia crece linealmente con N. Con indice (Tier 1+), crece logaritmicamente.
  • RAM de Qdrant se puede reducir con on_disk=true (~100 MB para 60K vectores), pero sigue siendo significativamente mayor que LanceDB.
  • Binary quantization de Qdrant (32x compresion, 40x mas rapido) es unica y muy util para vectores de alta dimension (>1024 dims) normalizados.
  • Datos de benchmarks: Brian Sunter (2025, independiente), Qdrant oficial (2024, 8 vCPU/32GB), LanceDB produccion (Sprytnyk, 2025, 700M vectores).

Recomendacion para ai_assistant

  • Default (ahora): Tier 0 — sin cambios. Funciona para la mayoria de usuarios.
  • Primer upgrade: Tier 2 — LanceDB. Un feature flag (rag-lance), sin dependencias externas, mejora x100 en rendimiento. Esta es la recomendacion principal.
  • Si se necesita escala corporativa: Tier 3 — Qdrant self-hosted. Solo si el usuario ya tiene Docker/infra.
  • Evitar: Pinecone (vendor lock-in total, no self-hosteable, el mas caro).
