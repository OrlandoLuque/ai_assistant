Qué es esta matriz

Esta tabla cruza casos de uso (columnas) con tecnologías/módulos (filas) del crate ai_assistant. Cada celda indica si esa tecnología está disponible, activa por defecto o requiere configuración para ese caso de uso concreto.

Perspectiva "configurable": la matriz muestra todo lo que se puede activar al configurar el asistente, no solo lo que se usa típicamente. Si una tecnología puede añadirse a un chat básico habilitando un feature flag, aparece como Feature, no como N/A.

Estados de las celdas

EstadoColorSignificadoAcción del desarrollador
Core Azul Siempre activo, no se puede desactivar. Forma parte del flujo base del asistente. Ninguna — se usa automáticamente al llamar a send_message() o similar.
Auto Violeta Se activa automáticamente si el módulo está compilado. No requiere llamada explícita. Puede deshabilitarse con configuración explícita si no se desea.
Available Verde Compilado siempre (sin feature flag), pero el caller debe usarlo explícitamente. Importar el módulo y llamar a su API. Ej: RequestSigner::new().
Feature Naranja Requiere activar un feature flag en Cargo.toml para compilarse. Añadir el flag: cargo build --features "rag", luego usar la API.
N/A Gris Técnicamente imposible o sin sentido para este caso de uso. Pasa el ratón por encima para ver el motivo. No aplica.
Editado Borde rojo + punto Celda modificada por el usuario con una propuesta o pregunta. Se incluirá en el prompt generado.

Opacidad: típico vs. no típico

Dentro de cada fila, las celdas tienen dos niveles de opacidad:

  • Opacidad completa — Esta tecnología se usa normalmente en este caso de uso. Es lo esperado cuando se configura ese tipo de asistente.
  • Opacidad reducida (dimmed) — La tecnología está disponible y se puede activar, pero no es típica para este caso de uso. El desarrollador puede habilitarla si lo desea, pero no es lo habitual.

Las celdas N/A nunca se atenúan — siempre aparecen en gris. Al pasar el ratón por una celda atenuada, se ilumina para facilitar la lectura.

Grupos de tecnologías (filas)

Las filas están organizadas en 12 grupos colapsables:

  • Core & Providers — Providers de LLM, config, sesiones, HTTP client, AWS Bedrock SigV4, provider plugins (7 presets), model registry, EmbeddingProvider trait (4 backends), GeminiProvider nativo, AsyncProviderPlugin (sync/async bridge), Ollama tool calling nativo, PromptToolFallback
  • RAG & Knowledge — RAG, embeddings, graphs, documentos (CSV, PPTX, XLSX, EML), feeds, KPKG, pgvector, semantic chunking
  • Security & Access Control — Sanitización, PII, moderación, RBAC, cifrado, SHA-256/HMAC signing, GuardrailPipeline (6 guards)
  • Tools & Agent — Tool registry, agentic loop, MCP, prompt chaining
  • Autonomous Agent — Loop autónomo, políticas, sandbox, OS tools, scheduler, butler (con detección de speech), browser CDP, code sandbox, container tools
  • Multi-Agent & Orchestration — Orquestador, memoria compartida, task decomposition, agentes distribuidos
  • Containers & Documents — ContainerExecutor (Docker/bollard), ContainerSandbox (ejecución aislada), SharedFolder (bind mounts + cloud sync), DocumentPipeline (PDF, DOCX, PPTX via pandoc)
  • Audio & Speech — SpeechProvider trait, STT (OpenAI Whisper, whisper.cpp local, Google Cloud), TTS (OpenAI, Google, Piper, Coqui), factory (6 providers)
  • Networking & Distribution — DHT, CRDT, MapReduce, QUIC, Merkle sync, P2P manager
  • HTTP Server & API — REST server, SSE, WebSocket, resumable streaming, OpenAPI, webhooks, colas
  • Analytics & Evaluation — Métricas, analytics, confianza, alucinaciones, ensembles, A/B testing, LLM-as-Judge (6 criteria, pairwise, RAG faithfulness), forecasting
  • Cross-Cutting Utilities — Adaptive thinking, memoria, event bus, caching, plugins, i18n, OTel, cloud connectors, WASM, smart suggestions, CI/CD pipeline, etc.

La etiqueta [nombre-flag] junto al nombre de una tecnología indica qué feature flag la activa.

Cómo usar la herramienta de edición

1. Revisar celdas

Haz clic en cualquier celda para abrir el panel de edición. Verás:

  • El estado actual de la celda y su feature flag (si tiene)
  • Notas del sistema sobre el módulo
  • Si es N/A, el motivo técnico específico

2. Proponer cambios

En el popup puedes:

  • Cambiar estado: seleccionar un nuevo estado (Core, Auto, Available, Feature, N/A)
  • Propuestas: "debería ser Core", "debería auto-activarse", "debería ser configurable", "añadir soporte", "eliminar"
  • Preguntas: marcar como "Pregunta/duda" y escribir la consulta
  • Texto libre: instrucciones, contexto o cualquier comentario

Las celdas editadas se marcan con borde rojo y un punto indicador.

3. Generar prompt

Cuando hayas terminado de revisar, pulsa "Generar Prompt". Se creará un texto estructurado con todos tus cambios, propuestas y preguntas, listo para copiar y pegar en una conversación con el asistente (Claude, etc.) para que aplique los cambios en el código.

4. Filtros

Usa la barra de filtros para mostrar solo filas con un tipo de celda concreto. El filtro "Editados" muestra solo las filas donde has hecho cambios.

Atajos de teclado

  • Esc — Cerrar popup
  • Ctrl+Enter — Guardar edición
