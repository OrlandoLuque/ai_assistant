Three Ways to Deploy

Same crate, different feature flags. Pick the deployment model that fits your needs.

Tier 1

Embedded Library

Add ai_assistant as a dependency in your Cargo.toml. Use only the features you need.

  • Selective feature flags (pay only for what you use)
  • Direct Rust API integration
  • Compile-time dead code elimination
  • Ideal for Rust applications, CLI tools, desktop apps
  • Embeddable in egui, Tauri, or any Rust binary
Tier 2 — Most Common

Standalone Server

Build the included HTTP server binary. Get a full REST + WebSocket API with a single cargo build.

  • OpenAI-compatible chat API
  • SSE streaming, gzip compression
  • Auth middleware, RBAC, rate limiting
  • TLS (rustls), CORS, Prometheus metrics
  • Graceful shutdown, request correlation IDs
  • Dockerfile included
Tier 3

Distributed Cluster

Enable distributed-network and p2p features. Nodes discover each other and sync automatically.

  • QUIC/TLS 1.3 transport (Quinn)
  • Kademlia DHT for peer discovery
  • CRDTs for conflict-free replication
  • Anti-entropy Merkle sync
  • Reputation tracking, join tokens
  • No Redis, no Kafka, no ZooKeeper

Security at Every Layer

Not bolted on as an afterthought. Security is woven into the architecture from input validation to encrypted persistence.

🛡

Constitutional AI Guardrails

Configurable input/output guardrail pipeline with streaming support. Block, warn, or redact.

🔍

PII Detection

Real-time detection of emails, phones, SSNs, credit cards, and custom patterns. Auto-redaction on output.

🔒

RBAC Access Control

Role-based access with MFA, CIDR IP ranges, usage limits, and time-based policies.

🔐

AES-256-GCM Encryption

Content encryption at rest with authenticated encryption. Key management built in.

🔗

mTLS & QUIC

Mutual TLS for node-to-node communication. QUIC transport with TLS 1.3 for distributed clusters.

🧪
